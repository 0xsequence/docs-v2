# Authentication Providers

Sequence Auth combined with web2 authnetication enables developers to bring web3 benefits to everyday web2 users

In order to setup your Embedded Wallet instance, you need to obtain a `client id` from the login provider of your choice, and, enable the Allowed Origins function to authorize your application, so someone else doesn't also use your client id

Finally, you'll need to set a `Recovery Wallet` with a password.

The following steps outlined how to accomplish these neccessary configuration:

## 1. Choose your Web2 Provider

### Google Authentication

First, login to the [Google Developer Console](https://console.cloud.google.com/) and create a project and billing.

![embedded wallet google client id](/img/embedded_wallet/embedded_wallet_google_client_id_creation.png)

Then create a `Web application` and `name` your `OAuth 2.0 client`

And add any allowed origins required for your application to function

![embedded wallet allowed origins](/img/embedded_wallet/embedded_wallet_allowed_origins.png)

Then, create your credentials.

Once created, you will see a confirmation screen for the OAuth client, then copy the `Client ID` to be used in [Step 2](/solutions/wallets/embedded-wallet/examples/authentication-providers#update-login-providers)

### Apple Authentication

Login to the [Apple Console](https://appleid.apple.com/)

## 2. Update Embedded Wallet Config

Navigate to Embedded Wallet Section in [Sequence Builder](https://sequence.build)

![embedded wallet navigate to page](/img/embedded_wallet/embedded_wallet_navigate.png)

### Update Login Providers
Select the `Login Providers` `Edit` button to update the providers from [Step 1](/solutions/wallets/embedded-wallet/examples/authentication-providers#1-choose-your-web2-provider)

![embedded wallet select Login providers](/img/embedded_wallet/select_login_providers.png)

Click on the `+ Add new sign-in option` and select the provider 

![embedded wallet login providers](/img/embedded_wallet/login_providers.png)

Then input the `Client ID` from [Step 1]() and click save changes

![input login provider client id](/img/embedded_wallet/input_login_provider_client_id.png)

### Update Allowed origins

Select the `Allowed Origins` `Edit` button
![select allowed origins button](/img/embedded_wallet/select_allowed_origins.png)

Add a domain and click `Save changes`

![add a domain](/img/embedded_wallet/add_a_domain.png)

### Update Recovery Wallet
Then update the `Recovery Wallet` by selecting the `Set wallet` button

![set recovery wallet](/img/embedded_wallet/set_recovery_wallet.png)

Then input your recovery wallet and set a password, noting it to write it down somewhere.

![input recovery wallet](/img/embedded_wallet/input_recovery_wallet.png)

Finally, create the configuration

![create configuration](/img/embedded_wallet/create_configuration.png)

And copy the `WaasConfigKey` to use in your project
